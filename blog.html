Q1.What is the difference between local storage and session storage?

Ans:Difference between localand session storage:

SessionStorage is very similar to localStorage ; the difference is that while data in localStorage doesn't expire, data in sessionStorage is cleared when the page session ends. Whenever a document is loaded in a particular tab in the browser, a unique page session gets created and assigned to that particular tab. Session storage gets deleted if we reload the page or close the tab but local storage doesnot get deleted if we reload.

Q2.What is the difference between global scope and block scope? ?

Ans:Difference between global scope and block scope:

Global scope is controlled by the window object while in Node. js, it's controlled by the global object. Block scopes are what we get when you use if statements, for statements. For instance if we declare a variable globally outside any block it will be accessible from within any block scope but variables declared in block scopes will not be accessible globally.

Q3. How does the event loop work?

Ans:Event loop:

To understand event loop we need to understand 3 things , which are: Call stack, web APIs and task queue. As javascript is a single threaded language , when it does something which takes comparatively long time than other codes it can blocks the browsers functionality and while running those asynchronus codes browser cant do anything else. Callstack is a data structure which keeps the track of the code that are being executed. When we run an asynchronus code , to avoid the browsers blocking behavior we pass the async code from the callstack to browsers webAPI and the async code executes there and the result is passed to the task queue. The eventloop loop basically checks the callstack if it is empty or not , if it is empty it takes the async codes result and passes it to stack and waits for the stack to clear and then pass another result fromo taskque(if any).
